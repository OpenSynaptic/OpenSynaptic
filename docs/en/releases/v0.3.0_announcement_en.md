### 2. Intelligent ID Lease System (★★★★☆ Major Feature)

**v0.3.0 New:**

A complete adaptive ID management system solving device mobility and connection lifecycle:

```json
"security_settings": {
  "id_lease": {
    "offline_hold_days": 30,
    "base_lease_seconds": 2592000,
    "high_rate_threshold_per_hour": 60,
    "ultra_rate_threshold_per_hour": 180,
    "adaptive_enabled": true,
    "metrics_emit_interval_seconds": 5
  }
}
```

**Core Capabilities:**

1. **Smart ID Allocation**
   - Auto-assign unique ID on first connection
   - ID caching enables recovery on reconnect (data continuity)

2. **Adaptive Lease Shortening**
   - Real-time device connection rate monitoring
   - High-rate (60+/hour) → shorten to 20% of base
   - Ultra-rate (180+/hour) → immediate release (0 sec lease)
   - Auto-handle high-churn scenarios

3. **Offline Device Handling**
   - Auto-release after 30 days offline
   - Full reconnection and ID recovery support
   - Visual metrics and performance monitoring

4. **3 Pre-configured Scenarios**
   - High-churn IoT network
   - Industrial monitoring (low churn)
   - Development/testing environment

## 🚀 Usage Examples

### 1. Enable ID Lease System

```json
// Config.json
{
  "security_settings": {
    "id_lease": {
      "offline_hold_days": 30,
      "base_lease_seconds": 2592000,
      "high_rate_threshold_per_hour": 60,
      "ultra_rate_threshold_per_hour": 180,
      "adaptive_enabled": true,
      "metrics_emit_interval_seconds": 5
    }
  }
}
```

### 2. Use ID Lease API

```python
from opensynaptic.services.id_lease_system import IDLeaseManager

# Initialize
lease_mgr = IDLeaseManager(config_path="Config.json")

# Get or allocate ID
device_id = lease_mgr.get_or_allocate_id(device_identifier="sensor_01")

# Set metrics callback
def on_metrics(metrics):
    print(f"Active devices: {metrics['active_count']}")
    print(f"Pending release: {metrics['pending_release_count']}")

lease_mgr.set_metrics_sink(on_metrics)

# Handle device activity
lease_mgr.record_device_activity(device_id)
```

### 3. Bidirectional Communication

```python
from opensynaptic.core.transport_layer.drivers import udp

# Send data
udp.send(b"sensor_data", {"host": "192.168.1.100", "port": 5000})

# Receive data
def on_data(data, addr):
    print(f"Received from {addr}: {data}")

udp.listen({"host": "0.0.0.0", "port": 5000}, on_data)
```

## ⚠️ Known Issues

### 1. IDE Parse Cache
**Symptom:** Stale diagnostics after code edit  
**Cause:** Language service cache  
**Workaround:** IDE restart or manual reindex

### 2. Rust Native Dependency
**Symptom:** `rscore` commands fail  
**Cause:** Local Rust toolchain or build issues  
**Solution:** See `docs/releases/v0.2.0.md`

### 3. Windows Multi-Process Scheduling
**Symptom:** Benchmark result variance  
**Cause:** Windows scheduler and CPU affinity  
**Workaround:** Use fixed process/thread combinations

### 4. ID Lease Persistence
**Symptom:** Startup failure on data corruption  
**Cause:** `data/id_allocation.json` format issue  
**Recovery:** Delete file and reinitialize
